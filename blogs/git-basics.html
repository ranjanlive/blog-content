What is Git?

Git is a distributed version control system that helps you track changes in your code. It's essential for both solo developers and teams working on projects together.

Basic Git Commands

1. Initialize a Repository

git init

This command creates a new Git repository in your current directory.

2. Clone a Repository

git clone https://github.com/username/repository.git

Downloads a copy of a remote repository to your local machine.

3. Basic Workflow Commands

git status # Check repository status git add filename # Add file to staging area git add . # Add all files to staging area git commit -m "message" # Commit changes git push # Push changes to remote git pull # Pull changes from remote
Tip: Always check your repository status with git status before making commits.

Branching

git branch # List branches git branch branch-name # Create new branch git checkout branch-name # Switch to branch git checkout -b branch-name # Create and switch to new branch git merge branch-name # Merge branch into current branch

Best Practices

  • Write clear, descriptive commit messages
  • Commit frequently with logical changes
  • Pull before pushing to avoid conflicts
  • Use branches for new features
  • Keep your main/master branch stable

Conclusion

These basic Git commands will help you get started with version control. Practice these commands regularly, and they'll become second nature in your development workflow.
